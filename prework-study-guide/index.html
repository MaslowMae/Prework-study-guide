HTML

  • We use HTML to build a sturdy frame, using HTML elements as our frames building blocks.
  • The DOCTYPE html is a declaration that tells the browser what document type it should expect when it is opened.
  • The opening tag is HTML which represents the root or start of the document.
  • The lang attribute is found inside the opening tag which tells the browser the language for the page.
  • The head element contains information about the webpage such as metadata to render the page correcly and the title of the page for SEO.
  • The body element represents the visible content shown to the user for them to also interact with.
  • The section element is inside the body to provide organized thematic content.
  • The header element is at the top of the body and can contain navigation links or menus.
  • The footer element is at the bottom of the page and usually contains author, copyright, contacts, or sitemap.

Git

  • Git allows us to work locally and change our work via branching in a way that doesn't conflict with collaborators
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch
  • stage changes when done with git add -A
  • commit the changes with git commit -m "added blank"
  • git init: initiallizes an existing directory as a git repository
  • when in the correct directory in the terminal, type code . to open visual studio code workspace
  • an SSH key is an access credential for the secure shell network protocol and allow for access without repeated password input
  • it is important to properly create and share your ssh keys to allow developers to interface with remote services like git
    • the ssh keys are generated and then added to the ssh agent which keeps track of keys and passphrases
